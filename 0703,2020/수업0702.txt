String buffer 나  StringBuilder는 사용법이 유사하다. 


<<상속>>
상속관계에 있는 자식클래스는 부모의 모든 속성과 동작을 자동으로 갖게되며 사용 할 수 있음
그러나 만약 부모로부터 물려받은 동작중에
그동작의 내용이 자신에게 맞지 않으면
자신에게 맞도록 다시 정의하여 사용할 수있음
이것을 메소드의 재정의 (method overriding) 이라고 한다
메소드 재정의 를 위해서는 메소드 이름뿐만이 아니라
매개변수의 개수와 자료형까지 일치하게 재정의 해야함 

<<상속>>
자바에서는 새로운 클래스를 만들때에 이미만들어진 클래스를 확장하여 새로운 클래스를 
만들수도 있다. 
이와같이 의미있는 클래스를 확장하여 새로운 클래스를 만드는 것을 상속이라고 한다. 

자바는 이미 4000개가 훨씬넘는 프로그램에 도움이 될 클래스를 제공하고 있다. 
그 중에 object 라는 클래스는 모든 자바 클래스둘의 최상위 클래스이다. 
그리고 내가 만드는 모든 클래스들도 묵시적으로 object 의 후손이다. 
class Person {
}
과 같이 클래스를 만들면 의미상 다음과 같은구죠
class Person extends

 자바의 모든 클래스들읜 Object 클래스의 후손이다/
그래서 모든 클래스 드은 Object 클래스의 메소드들을 자유롭게 사용할수 있다.
그래고 필요하다면 자신에게 맞도록 재정의(overriding) 할 수 있다/

Object 클래스에서 오버라이딩 하는 대표적인 메소드가 
equals()와 toString() 입니다/
equals() 는 두개의 객체가 서로 동일한 값으로 구성되어있는지
판별하기 위한 용도로 사용되며 

만약에 equals ()를 오버라이딩 안하면 == 하고 똑같은 결과를 반환한다!!!

toString()은 그 객체의 속성값들을 문자열로 반환하는 용도로 사용이 된다. 
toString()을 오버라이딩 하지 않으면 그 객체가 바라보고있는 메모리상의 주소값을 반환한다.

상속관계에 있을떄에 상위클래스의 참조변수는 
자식클래스의 객체를 참조할 수 있다. 

=======================================================================
상위클래스의 참조변수는 자식클래스의 객체를 참조할 수 있다. 
그러나 자식클래스에서 추가된 멤버에는 접근할 수 없습니다. 
자식클래스 자료형으로 형변환 한 후에 접근할 수 있다. 
이때 어떠한 자식 클래스인지 판별하기 위해서는 instanceof  연산자를 이용한다.
